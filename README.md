# Scratch Score

A native iOS golf scoring app that tracks rounds with the **scoring‑zone method** instead of raw stroke counting, to help you make better on‑course decisions and score lower.

Each hole is recorded as:

| Row | Meaning |
|-----|---------|
| **A** | Shots to *enter the scoring zone* (get within 100 yds of the green) |
| **B** | Shots from the scoring zone to *hole out* (goal ≤ 3) — **includes putts** |
| **C** | Total = **A + B** (auto‑calculated, never stored) |
| **D** | Total putts — a breakdown of B, tracked for stats, **never re‑added** into the total |

Plus per‑hole markers: penalty strokes, up‑&‑down attempt/made, putt made over 4 ft.

## Stack

- **SwiftUI**, iOS 17+ · **SwiftData** (offline‑first local store) · **Swift Charts**
- **Supabase** — Auth (Apple / Google / email) + Postgres + Edge Functions + cloud sync
- Project generated by **XcodeGen** (`project.yml`); the `.xcodeproj` is a build artifact (git‑ignored)

Architecture notes live in the code (`ScratchScore/Core/Sync/SyncEngine.swift` is the highest‑risk logic) and in the plan under `~/.claude/plans/`.

## Prerequisites

- Xcode 16+ with the iOS platform installed (Xcode → Settings → Components)
- Homebrew tools: `brew install xcodegen supabase/tap/supabase`
- A [Supabase](https://supabase.com) project (free tier is fine)
- Apple Developer Program membership (for Sign in with Apple, TestFlight, submission)

## Setup

### 1. Backend (Supabase)

```bash
supabase link --project-ref <YOUR_PROJECT_REF>
supabase db push                     # applies supabase/migrations/0001_init.sql (schema + RLS)
supabase functions deploy delete-account
```

Set the Edge Function secrets (never shipped in the app):

```bash
supabase secrets set \
  APPLE_TEAM_ID=XXXXXXXXXX \
  APPLE_CLIENT_ID=app.scratchscore \
  APPLE_KEY_ID=XXXXXXXXXX \
  APPLE_PRIVATE_KEY="$(cat AuthKey_XXXXXXXXXX.p8)"
```

In the Supabase dashboard → **Authentication → Providers**:
- **Apple**: enable; add the app **Bundle ID** (`app.scratchscore`) to authorized client IDs.
- **Google**: enable with a Google Cloud **Web** OAuth client id/secret.
- **URL Configuration → Redirect URLs**: add `app.scratchscore://login-callback`.

### 2. App secrets

```bash
cp ScratchScore/App/Secrets.example.xcconfig ScratchScore/App/Secrets.xcconfig
```

Edit `Secrets.xcconfig` with your Supabase URL and **publishable (anon)** key. (This file is git‑ignored. The URL's `//` is escaped as `https:/$()/…` — keep that form.)

### 3. Generate & open

```bash
xcodegen generate
open ScratchScore.xcodeproj
```

Set your **Development Team** on the `ScratchScore` target (Signing & Capabilities). The **Sign in with Apple** capability is already declared in `ScratchScore/Supporting/ScratchScore.entitlements`.

> Until `Secrets.xcconfig` holds real values the app launches into a "Backend not configured" screen instead of crashing — handy for UI work.

## Build & run

```bash
# In Xcode: select an iPhone simulator and press ⌘R, or:
xcodebuild -project ScratchScore.xcodeproj -scheme ScratchScore \
  -destination 'platform=iOS Simulator,name=iPhone 16' build
```

## Tests

```bash
xcodebuild -project ScratchScore.xcodeproj -scheme ScratchScore \
  -destination 'platform=iOS Simulator,name=iPhone 16' test
```

`ScratchScoreTests/ScoringLogicTests.swift` verifies the core rule that **C = A + B and putts are never re‑added**, plus round‑stat aggregation.

## App Store submission checklist

- [x] Sign in with Apple (required because Google sign‑in is offered)
- [x] In‑app account deletion (Settings → Delete Account), no support‑flow gating
- [x] Apple token revocation on delete (Edge Function → `appleid.apple.com/auth/revoke`)
- [x] Launch screen (Info.plist `UILaunchScreen`), app icon (1024², no alpha)
- [x] `ITSAppUsesNonExemptEncryption = false`
- [x] Row Level Security on every table; secret key only in the Edge Function
- [ ] Host `docs/PRIVACY.md` and `docs/TERMS.md`; set the URLs in App Store Connect and update `LegalLinks` in `SignInView.swift`
- [ ] Complete App Privacy nutrition labels (collects email + name + user content for App Functionality; no tracking)
- [ ] Provide a working **email/password demo account** in App Review notes (reviewers can't use your Apple/Google)
- [ ] Screenshots, description, keywords, support URL, age rating → TestFlight → submit
